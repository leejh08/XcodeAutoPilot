🛠️ Xcode 안 켜고 iOS 개발하는 MCP 서버를 만들었습니다

iOS 개발하면서 이런 경험 있으시죠?

빌드 에러 → AI한테 붙여넣고 → 수정 받고 → Xcode 가서 빌드해보고 → 또 에러 나오면 다시 반복...
UI 바꾸고 → 시뮬레이터 켜고 → 확인하고 → 다시 코드로 돌아오고...

저는 이 과정 자체를 없애고 싶었습니다.

그래서 만들었습니다. **XcodeAutoPilot** — AI가 직접 빌드를 돌리고, 에러를 분석하고, 코드를 수정하고, 다시 빌드해서 에러가 없어질 때까지 혼자 반복하는 MCP 서버입니다.

---

**개발자가 할 일:**

"이 프로젝트 빌드 에러 고쳐줘" — 딱 한 마디면 끝입니다.

나머지는 AI가 알아서 합니다.

1. 빌드 실행 → 에러 수집
2. 에러가 발생한 코드 + 관련 함수 정의 + 호출부까지 분석
3. 수정 사항 적용 (백업 후)
4. 다시 빌드 → 에러가 남아있으면 처음부터 반복
5. 에러 0개가 되면 완료

수정이 오히려 에러를 늘리면 자동으로 롤백합니다.

---

**UI도 AI가 직접 확인합니다.**

"버튼 색상 파란색으로 바꾸고 잘 됐는지 확인해줘" — 이 한 마디로

AI가 시뮬레이터에서 앱을 빌드하고 실행한 뒤 스크린샷을 직접 찍어서 눈으로 확인합니다. Xcode를 열 필요가 없습니다.

---

**다른 비슷한 도구와 차이점:**

빌드 에러 메시지만 AI에게 넘기는 게 아니라, 에러가 발생한 함수 전체와 그 함수를 호출하는 코드까지 함께 넘깁니다. AI가 전체 맥락을 보고 첫 시도에 정확하게 고칠 수 있도록요.

Tuist 프로젝트도 지원하고, Claude Code뿐만 아니라 Gemini CLI, Codex CLI 등 MCP를 지원하는 모든 AI CLI에서 사용할 수 있습니다.
